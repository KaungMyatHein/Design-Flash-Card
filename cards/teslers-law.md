# English: Tesler's Law (Law of Conservation of Complexity)
# Myanmar: Tesler's Law (Complexity မပျောက်ဆုံးနိုင်ဘူး ဆိုတဲ့ ဥပဒေသ)

## Question
English: What does Tesler's Law say about complexity in product design, and what should designers do about it?

## Question_MM
Tesler's Law က product design ထဲက complexity အကြောင်း ဘာပြောထားလဲ၊ designer တွေက ဘယ်လိုလုပ်သင့်လဲ?

## Answer
English: Tesler's Law states that every system has a certain amount of complexity that cannot be removed — it can only be moved around. A designer's job is to absorb as much of that complexity as possible so the user doesn't have to deal with it.

## Answer_MM
Tesler's Law က system တိုင်းမှာ ဖယ်လို့မရတဲ့ complexity ပမာဏ ရှိတယ်လို့ ဆိုတယ်။ ဒီ complexity က ပျောက်သွားတာ မဟုတ်ဘူး — နေရာပြောင်းတာပဲ။ ဒါကြောင့် designer ရဲ့ အလုပ်က ဒီ complexity ကို ကိုယ့်ဘက်ကနေ အများဆုံး ယူပြီး user ဘက်ကို အနည်းဆုံးပဲ ရောက်အောင် လုပ်ပေးရတာ။

## Why
- English: Complexity never disappears — when you simplify one part, the complexity shifts to another part (often the backend or design system)
- English: If designers don't handle the complexity, users are forced to figure things out themselves, leading to frustration
- English: Consistency and predictability are key tactics — they help users learn the system faster without needing to memorize rules
- English: Over-simplification can backfire too — hiding too much makes the interface feel like a black box where users lose control
- English: The goal is not zero complexity, but putting complexity in the right place — behind the scenes, not in front of the user

## Why_MM
- Complexity ဆိုတာ ပျောက်သွားတာ မရှိဘူး — တစ်နေရာ simple ဖြစ်ရင် နောက်တစ်နေရာမှာ complex ဖြစ်သွားတယ်
- Designer က မကိုင်တွယ်ရင် user ကိုယ်တိုင် ရှာကြံရပြီး frustrate ဖြစ်တယ်
- Consistency နဲ့ predictability က user ကို system ကို မြန်မြန်သင်ယူနိုင်အောင် ကူပေးတယ်
- ဒါပေမယ့် အရမ်း simple လုပ်လွန်းရင်လည်း user က control လုပ်လို့ မရတော့ဘူးလို့ ခံစားရတယ်
- ရည်ရွယ်ချက်က complexity ကို zero ဖြစ်အောင် မဟုတ်ဘူး — user မမြင်ရတဲ့ နေရာမှာ ထားဖို့

## Example
- English: Email apps like Gmail handle complex filtering and sorting behind the scenes so users just see a clean inbox
- English: Google Search processes billions of pages but users only need to type a few words — the complexity is absorbed by the system
- English: Ride-hailing apps like Grab hide GPS routing, driver matching, and fare calculation behind a simple "Book" button
- English: Auto-save in Google Docs removes the complexity of remembering to save — the system handles it silently

## Example_MM
- Gmail က email filtering, sorting အကုန် နောက်ကွယ်မှာ လုပ်ပေးတယ် — user က clean inbox ပဲ မြင်ရတယ်
- Google Search မှာ user က စကားလုံးနှစ်လုံးသုံးလုံးပဲ ရိုက်ရတယ်၊ billions of pages ကို process လုပ်တာက system ဘက်မှာ
- Grab app မှာ GPS routing, driver matching, fare calculation အကုန်လုံးကို "Book" button တစ်ခုနောက်မှာ ဖျောက်ထားတယ်
- Google Docs က auto-save လုပ်ပေးတယ် — save ဖို့ မှတ်ရတဲ့ complexity ကို system က ယူသွားတယ်
