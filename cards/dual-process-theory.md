# English: Dual Process Theory (System 1 & System 2)
# Myanmar: Dual Process Theory (စဉ်းစားမှု စနစ် ၂ မျိုး)

## Question
English: What are System 1 and System 2 thinking, and why do they matter in UX design?

## Question_MM
User တွေ ဆုံးဖြတ်ချက်ချတဲ့အခါ သူတို့ရဲ့ ဦးနှောက်က ဘယ်လို စဉ်းစားမှု ၂ မျိုးနဲ့ အလုပ်လုပ်တာလဲ၊ UX designer အနေနဲ့ ဘာကြောင့် သိဖို့လိုတာလဲ?

## Answer
English: Our brain uses two thinking systems. System 1 is fast, automatic, and gut-driven — like smiling when you see a baby. System 2 is slow, deliberate, and effortful — like solving a math problem. Most daily decisions happen through System 1, so UX designers should design for quick, intuitive interactions rather than forcing users into heavy thinking.

## Answer_MM
လူတွေရဲ့ ဦးနှောက်မှာ စဉ်းစားတဲ့ပုံစံ ၂ မျိုးရှိတယ်။ System 1 က မြန်တယ်၊ အလိုအလျောက်ဖြစ်တယ်၊ gut feeling နဲ့သွားတယ် — ကလေးလေးတစ်ယောက်ကို မြင်ရင် မသိမသာ ပြုံးလိုက်မိသလိုမျိုးပေါ့။ System 2 ကတော့ နှေးတယ်၊ အားထုတ်ရတယ်၊ သေသေချာချာ စဉ်းစားရတယ် — ဂဏန်းတွက်ရသလိုမျိုးပေါ့။ နေ့စဉ် ဆုံးဖြတ်ချက်အများစုက System 1 နဲ့ ဖြစ်နေတာမို့ UX designer တွေက user ကို အများကြီး စဉ်းစားခိုင်းတာထက် intuitive ဖြစ်အောင် design လုပ်ဖို့ လိုတယ်။

## Why
- English: Users don't have time or energy to carefully analyze every interaction — System 1 handles most daily decisions automatically
- English: When design requires too much conscious effort (System 2), users feel frustrated and are more likely to abandon tasks
- English: Gut reactions and first impressions are formed by System 1 within milliseconds, shaping how users perceive your product
- English: Designing for System 1 means using familiar patterns, clear visual cues, and reducing cognitive friction
- English: Understanding when to engage System 2 (like for important confirmations) helps prevent costly user errors

## Why_MM
- User တွေက interaction တိုင်းကို သေချာ analyze လုပ်နေဖို့ အချိန်ရော energy ရော မရှိဘူး — နေ့စဉ် ဆုံးဖြတ်ချက်အများစုကို System 1 က auto ကိုင်တွယ်တယ်
- Design က conscious effort (System 2) အများကြီး တောင်းဆိုရင် user တွေ frustrate ဖြစ်ပြီး task ကို ပစ်ထားခဲ့တတ်တယ်
- Gut reaction နဲ့ first impression က System 1 ကနေ millisecond အတွင်းမှာ ဖြစ်တယ်၊ product ကို ဘယ်လိုမြင်မလဲဆိုတာ အဲ့မှာ ဆုံးဖြတ်သွားတယ်
- System 1 အတွက် design လုပ်တယ်ဆိုတာ familiar pattern တွေ၊ clear visual cue တွေသုံးပြီး cognitive friction ကို လျှော့ချတာပဲ
- ဘယ်အချိန်မှာ System 2 ကို engage လုပ်ရမလဲ သိရင် (important confirmation လိုမျိုး) user error ကြီးကြီးတွေကို ကာကွယ်နိုင်တယ်

## Example
- English: Amazon's 1-Click ordering leverages System 1 — no deliberation needed, just one tap to buy
- English: Google's simple search page lets System 1 take over — users type and hit enter without thinking about the interface
- English: Instagram's double-tap to like is a System 1 action — it feels so natural that users do it without conscious thought
- English: Banking apps use System 2 deliberately — asking users to confirm transfers with extra steps to prevent mistakes

## Example_MM
- Amazon ရဲ့ 1-Click ordering က System 1 ကို အသုံးချတယ် — စဉ်းစားစရာမလို၊ တစ်ချက်နှိပ်ရုံနဲ့ ဝယ်လို့ရတယ်
- Google ရဲ့ search page က ရိုးရှင်းလွန်းလို့ System 1 က take over လုပ်တယ် — user က interface အကြောင်း စဉ်းစားစရာမလိုဘဲ ရိုက်ပြီး Enter နှိပ်လိုက်ရုံပဲ
- Instagram မှာ double-tap နဲ့ like လုပ်တာက System 1 action ပဲ — သဘာဝကျလွန်းလို့ conscious thought မပါဘဲ လုပ်မိတယ်
- Banking app တွေကတော့ System 2 ကို တမင် သုံးတယ် — ငွေလွှဲတဲ့အခါ extra step တွေထည့်ပြီး confirm ခိုင်းတယ်၊ အမှားမဖြစ်အောင်ပေါ့
