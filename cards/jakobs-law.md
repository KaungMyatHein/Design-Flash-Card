# Jakob's Law
# Jakob ရဲ့ ဥပဒေသ

## Question
Why should you design interfaces that look and work like other popular sites?

## Question_MM
ဘာကြောင့် တခြား လူသုံးများတဲ့ site တွေနဲ့ ဆင်တူအောင် design လုပ်သင့်တာလဲ?

## Answer
Jakob's Law says users spend most of their time on OTHER sites, so they expect yours to work the same way. When you follow familiar patterns, people don't have to re-learn how to use your product.

## Answer_MM
User တွေက သူတို့ အချိန်အများစုကို တခြား site တွေမှာ သုံးနေတာ ဖြစ်တဲ့အတွက် မင်းရဲ့ site ကိုလည်း အဲ့လိုပဲ အလုပ်လုပ်မယ်လို့ မျှော်လင့်ထားတယ်။ Familiar pattern တွေ follow လုပ်ရင် user တွေ အသစ်ပြန်လေ့လာစရာ မလိုဘူး။

## Why
- Users build mental models from all the sites they use daily
- Breaking familiar patterns creates confusion and frustration
- Consistency across products reduces learning time dramatically
- Innovation should enhance familiar patterns, not replace them
- Users perceive familiar sites as more beautiful and trustworthy

## Why_MM
- User တွေက နေ့စဉ်သုံးတဲ့ site တွေကနေ mental model တွေ တည်ဆောက်ထားတယ်
- Familiar pattern တွေ ဖျက်လိုက်ရင် user တွေ ရှုပ်ထွေးပြီး စိတ်ပျက်တယ်
- Product တွေကြား consistency ရှိရင် learning time သိသိသာသာ လျော့တယ်
- Innovation က familiar pattern ကို အစားထိုးဖို့မဟုတ်ဘဲ ပိုကောင်းအောင်လုပ်ဖို့ ဖြစ်သင့်တယ်
- Familiar site တွေကို user တွေက ပိုလှတယ်၊ ပိုယုံကြည်ရတယ်လို့ ခံစားတယ်

## Example
- Search bar ကို page ရဲ့ ညာဘက်အပေါ်ထောင့်မှာ ထားတာ (Google, Amazon အားလုံး ဒီလိုပဲ)
- Shopping cart icon ကို header ရဲ့ ညာဘက်မှာ ထားတာ
- Logo ကို နှိပ်ရင် homepage ပြန်ရောက်တာ
- Hamburger menu ကို mobile app တွေမှာ ဘယ်ဘက်ထောင့်မှာ ထားတာ

## Example_MM
- Shopee Myanmar app မှာ search bar ကို ထိပ်ဆုံးမှာ ထားတယ် — user တွေ Lazada ကနေ switch လာလည်း ချက်ချင်း သုံးတတ်တယ်
- KBZ Pay app မှာ bottom tab bar pattern သုံးတယ် — Wave Money လိုပဲ ဖြစ်လို့ user အသစ်တွေ ရှုပ်စရာ မလိုဘူး
- Food Panda Myanmar မှာ cart icon ညာဘက်အပေါ်ထောင့် — Grab Food လိုပဲ ဖြစ်လို့ familiar ဖြစ်တယ်
- Myanmar online shop တွေမှာ Viber/Messenger link ထားတာ — user တွေ chat commerce ကို ကျင့်သားရနေပြီဖြစ်လို့
