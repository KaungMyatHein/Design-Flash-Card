# Feedback in Design
# Design မှာ Feedback (တုံ့ပြန်ချက်)

## Question
Why is feedback essential in interaction design?

## Question_MM
Interaction design မှာ feedback က ဘာကြောင့် မရှိမဖြစ် လိုအပ်တာလဲ?

## Answer
Feedback tells users what just happened after they take an action. It confirms whether a task was successful, failed, or is in progress. Without feedback, users feel lost and uncertain.

## Answer_MM
User က action တစ်ခု လုပ်ပြီးရင် ဘာဖြစ်သွားလဲ ပြောပြတာ။ အောင်မြင်လား၊ fail ဖြစ်လား၊ loading ဖြစ်နေသေးလား ပြောပြတယ်။ Feedback မရှိရင် user က ဘာဖြစ်နေလဲ မသိဘဲ ရပ်နေရတယ်။

## Why
- One of Don Norman's 7 fundamental design principles
- Best response time: as immediate as possible — within 0.1 seconds
- Long operations → show progress bars + time estimates
- Great feedback can turn negative into positive (Hootsuite owl sleeping)
- Lag kills momentum — like playing guitar with delayed notes

## Why_MM
- Don Norman ရဲ့ design အခြေခံမူ ၇ ခုထဲက တစ်ခု
- တုံ့ပြန်ချိန် တတ်နိုင်သမျှ ချက်ချင်း ဖြစ်ရတယ် — 0.1 စက္ကန့်အတွင်း
- ကြာတဲ့ task ဆို progress bar နဲ့ ခန့်မှန်းချိန် ပြပေးရတယ်
- Feedback ကောင်းရင် စိတ်ညစ်စရာကို ပျော်စရာ ပြောင်းနိုင်တယ် — Hootsuite ရဲ့ ဇီးကွက် "အိပ်သွားတာ" လို
- Lag ရှိရင် user ရဲ့ momentum ပျက်တယ် — guitar တီးပြီး note နောက်ကျမှ ထွက်လာသလိုမျိုး

## Example
- Button color change on click to confirm the action
- Loading spinner or progress bar during file uploads
- Success/error toast messages after form submission
- Hootsuite's owl animation sleeping during inactivity

## Example_MM
- Button နှိပ်ရင် အရောင်ပြောင်းပြီး "လုပ်ပြီးပြီ" ဆိုတာ ပြတယ်
- File upload နေတုန်း loading spinner ပတ်နေတယ်
- Form submit ပြီးရင် "အောင်မြင်ပါတယ်" / "မှားနေပါတယ်" toast ပေါ်လာတယ်
- Hootsuite မှာ user ဘာမှ မလုပ်ဘဲ ကြာရင် ဇီးကွက်လေး အိပ်သွားတယ်
