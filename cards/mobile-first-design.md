# Mobile-First Design
# Mobile-First Design (Mobile ဦးစားပေး ဒီဇိုင်း)

## Question
What is Mobile-First Design and why is it recommended?

## Question_MM
Mobile-First Design ဆိုတာ ဘာလဲ၊ ဘာကြောင့် recommend လုပ်တာလဲ?

## Answer
Mobile-First means designing for the smallest screen first, then progressively adding features and layout complexity for larger screens. It forces designers to focus on what truly matters.

## Answer_MM
Screen အသေးဆုံးအတွက် အရင် design ဆွဲပြီး screen ကြီးလာတာနဲ့ feature နဲ့ layout ထပ်ထည့်သွားတာ။ တကယ်အရေးကြီးတာကိုပဲ focus ချခိုင်းတယ်။

## Why
- Mobile traffic now exceeds desktop — mobile users come first
- Desktop-first then shrinking for mobile means painful feature cutting
- Mobile-first forces you to set clear content priorities
- Better performance — load only essentials, enhance on desktop
- Makes progressive disclosure feel natural

## Why_MM
- Mobile traffic က desktop ကို ကျော်သွားပြီ — mobile user ကို ဦးစားပေးရမယ်
- Desktop-first ဆွဲပြီး mobile အတွက် ချုံ့ရင် feature တွေ ဖျက်ရတာ နာကျင်တယ်
- Mobile-first က content priority ကို ရှင်းရှင်းလင်းလင်း ချမှတ်ခိုင်းတယ်
- Performance ပိုကောင်းတယ် — essential ပဲ load လုပ်ပြီး desktop မှာ enhance
- Progressive disclosure ကိုလည်း သဘာဝကျကျ ဖြစ်စေတယ်

## Example
- Designing a single-column layout for mobile, expanding to multi-column on desktop
- Hamburger menu on mobile that becomes a full nav bar on desktop
- Touch-friendly 44px minimum tap targets on mobile
- Hidden controls that reveal on hover (desktop) vs. always visible (mobile)

## Example_MM
- Mobile single-column → desktop multi-column layout ဖြစ်သွားတယ်
- Mobile မှာ hamburger menu → desktop မှာ nav bar အပြည့် ဖြစ်တယ်
- Mobile မှာ tap target 44px အနည်းဆုံး ထားတယ် (လက်ချောင်းနဲ့ ထိရလွယ်အောင်)
- Desktop မှာ hover မှ ပေါ်တာ → mobile မှာ အမြဲ မြင်ရအောင် ထားတယ်
