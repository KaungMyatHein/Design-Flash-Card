# Recognition Over Recall
# Recognition Over Recall (အမှတ်ရခြင်းထက် မြင်မှတ်ခြင်း)

## Question
What does "Recognition Over Recall" mean in UX?

## Question_MM
UX မှာ "Recognition Over Recall" ဆိုတာ ဘာလဲ?

## Answer
Users should recognize what to do from visible options rather than having to remember instructions from memory. Show choices instead of making users type or recall.

## Answer_MM
User ကို ဘာလုပ်ရမလဲ မှတ်မိခိုင်းမယ့်အစား မြင်ရတဲ့ ရွေးစရာတွေကနေ ရွေးခိုင်းတာ ပိုကောင်းတယ်။ မှတ်ခိုင်းတာထက် ပြပေးတာ ပိုလွယ်တယ်။

## Why
- Recognition is much easier than recall because it uses context clues
- Link labels should match the page heading they lead to — so users don't need to remember
- Cognitive load: recall costs more brain resources than visual or motor actions
- More clicks but less thinking = better experience

## Why_MM
- Recognition က context သဲလွန်စ သုံးတာမို့ recall ထက် အများကြီး လွယ်တယ်
- Link label နဲ့ ရောက်မယ့် page heading တူအောင်ထား — user မှတ်စရာ မလိုတော့ဘူး
- Recall က ဦးနှောက် resource ပိုကုန်တယ်၊ ကြည့်တာ/နှိပ်တာထက်
- Click ပိုများပေမယ့် တွေးစရာ နည်းရင် user ပိုပျော်တယ်

## Example
- Dropdown menus showing options instead of blank text fields
- Recently visited links changing color
- Search suggestions appearing as you type
- File browser showing icons with names instead of path-only navigation

## Example_MM
- ဗလာ text field အစား dropdown menu နဲ့ ရွေးစရာတွေ ပြတယ်
- ဝင်ပြီးသား link တွေ အရောင်ပြောင်းသွားတယ်
- ရိုက်နေတုန်း search suggestion ပေါ်လာတယ်
- File browser မှာ path ပဲ မဟုတ်ဘဲ icon လေးတွေနဲ့ name ပါ ပြတယ်
